🖼️ Mochi GUI 编程指南

扁平化 · 搭积木 · 线程安全

1. 概述

Mochi GUI 是内置于 Mochi 语言的图形界面库，基于 SDL2 和 SDL2_ttf，采用扁平化设计。它让您以“搭积木”的方式快速构建窗口、按钮、文本框、滑块、进度条、多行编辑器等界面组件。所有界面更新均在主线程安全执行，可无缝配合 Actor 并发模型。

使用 import("gui") 加载模块，无需额外安装脚本包。

2. 环境准备

安装依赖

Mochi GUI 需要 SDL2、SDL2_ttf 和 SDL2_gfx 开发库。

  • Linux (Ubuntu/Debian): 
    sudo apt install libsdl2-dev libsdl2-ttf-dev libsdl2-gfx-dev
  • macOS (Homebrew): 
    brew install sdl2 sdl2_ttf sdl2_gfx
  • Windows: 推荐使用 vcpkg 
    vcpkg install sdl2 sdl2-ttf sdl2-gfx
    或下载预编译的 `.lib` / `.dll` 文件。

编译 Mochi 解释器

在包含 GUI 模块的源码中，链接这些库：

g++ -std=c++17 -rdynamic -o mochi mochi.cpp -ldl -pthread -lSDL2 -lSDL2_ttf -lSDL2_gfx

确保 mochi.cpp 中已包含 GUI 模块的注册代码。

3. 基本概念

窗口 (Window)

所有界面的根容器。通过 gui.createWindow(title, width, height, style?) 创建。

层 (Layer)

窗口内可以添加多个层，用于组织控件。目前每个层独立，不影响渲染顺序（控件的 z 序与添加顺序一致）。

控件 (Widget)

按钮、标签、文本框、复选框、滑块、进度条、多行编辑器统称为控件。每个控件都有唯一的名称（name），通过 API 返回的代理对象访问。

代理对象 (Proxy)

创建控件后返回一个字典对象（代理），内含控件的内部 ID。所有操作（如 setText）都需要传入这个代理对象。

线程模型

所有 GUI 的实际绘制和事件处理都在 gui.run() 的主循环中执行。您在任意线程（包括 Actor）中调用 setText 等函数时，操作会被安全地投递到主线程执行，无需手动加锁。

4. 快速入门

loc gui = import("gui");

loc win = gui.createWindow("Hello GUI", 400, 300);
loc layer = gui.addLayer(win, "main");

loc btn = gui.addButton(layer, "btn1", "点我", 120, 100, 100, 40, fn() {
    print("按钮被点击！");
});

gui.run();   // 进入事件循环，程序阻塞于此

运行后会出现一个 400x300 的窗口，中间有一个蓝色圆角按钮，点击后控制台输出信息。点击窗口关闭按钮或调用 gui.quit() 可退出循环。

5. 控件详解

5.1 按钮 (Button)

gui.addButton(layer, name, text, x, y, w, h, callback?)

创建一个带文本的按钮。回调函数接收 0 个参数。

5.2 标签 (Label)

gui.addLabel(layer, name, text, x, y)

显示静态文本。宽度自动根据文字调整，高度固定 22px。

5.3 文本框 (TextField)

gui.addTextField(layer, name, text, x, y, w, h, onChange?)

单行文本输入框。支持鼠标点击定位、方向键、退格/删除、Home/End。可选 onChange 回调，每次文本变化时调用，传递当前文本字符串。

密码模式：gui.setPassword(tf, true) 可使输入显示为星号。

5.4 复选框 (CheckBox)

gui.addCheckBox(layer, name, text, x, y, w, callback?)

勾选框与文字标签的组合。回调函数接收一个布尔值参数表示当前状态。

5.5 滑动条 (Slider)

gui.addSlider(layer, name, min, max, step, x, y, w, callback?)

圆形滑块浮于轨道上。回调接收当前数值。可通过 gui.setValue(slider, 50) 动态设置。

5.6 进度条 (ProgressBar)

gui.addProgressBar(layer, name, x, y, w)

矩形进度条，默认范围 0~100。使用 gui.setValue(bar, 70) 更新，gui.setProgressRange(bar, min, max) 自定义范围。

5.7 多行文本框 (Memo)

gui.addMemo(layer, name, x, y, w, h)

功能类似记事本，支持键盘编辑、滚动、鼠标点击定位。通过 gui.getMemoText(memo) 和 gui.setMemoText(memo, text) 读写全部内容。

5.8 通用方法

方法说明
gui.setText(ctrl, text)设置控件文本（按钮、标签、文本框等）
gui.getValue(ctrl)获取滑动条/进度条的数值
gui.getText(ctrl)获取文本框/按钮/标签的文本
gui.isChecked(ctrl)获取复选框是否选中
gui.setEnabled(ctrl, bool)启用/禁用控件（变灰并忽略事件）
gui.setVisible(ctrl, bool)显示/隐藏控件
gui.setCallback(ctrl, fn)动态设置按钮或滑块回调
gui.setFocus(ctrl)将输入焦点移至文本框或 Memo
gui.setPassword(ctrl, bool)切换文本框密码模式
gui.setProgressRange(ctrl, min, max)设置进度条的最小/最大值
gui.quit()退出事件循环，关闭窗口

6. 布局系统

使用水平布局 (HLayout) 和垂直布局 (VLayout) 可以自动排列控件，无需手动计算坐标。

创建布局

loc hbox = gui.addHLayout(layer, "hbox1", x, y, w, h);
loc vbox = gui.addVLayout(layer, "vbox1", x, y, w, h);

布局属性

  • layout.setSpacing(n) – 设置子控件之间的间距（默认 6px）
  • layout.setPadding(n) – 设置容器内边距（默认 8px）

向布局添加控件

布局代理提供的方法**省略了 layer, name, x, y 参数**，其他参数与全局函数一致：

方法参数
hbox.addLabel(text)文本
hbox.addButton(text, w, h, callback?)文本、宽度、高度、可选回调
hbox.addTextField(text, w, h, onChange?)初始文本、宽度、高度、可选回调
hbox.addCheckBox(text, w, callback?)标签、宽度、可选回调
hbox.addSlider(min, max, step, w, callback?)最小值、最大值、步长、宽度、可选回调
hbox.addProgressBar(w)宽度
hbox.addMemo(w, h)宽度、高度

完整示例

loc gui = import("gui");
loc win = gui.createWindow("布局示例", 600, 400);
loc layer = gui.addLayer(win, "main");

loc vbox = gui.addVLayout(layer, "vbox", 10, 10, 200, 380);
vbox.setSpacing(8);
vbox.addLabel("用户登录");
vbox.addTextField("用户名", 160, 28);
vbox.addTextField("密码", 160, 28);
vbox.addCheckBox("记住我", 160);
vbox.addButton("登录", 160, 36, fn() { print("登录..."); });

loc memo = gui.addMemo(layer, "editor", 220, 10, 360, 380);
gui.run();

控件自动垂直排列，完美对齐。

7. 线程安全与 Actor

GUI 的所有实际更新都在主线程（事件循环）中执行。您可以在任意线程中调用 GUI 函数，它们会将操作推入任务队列，在下一帧处理。

结合 Actor 模型，可以轻松实现后台任务更新界面。

loc gui = import("gui");
loc win = gui.createWindow("Actor 更新界面", 300, 200);
loc layer = gui.addLayer(win, "main");
loc prog = gui.addProgressBar(layer, "bar", 50, 80, 200);

loc worker = spawn(fn(_) {
    loc i = 0;
    while (i <= 100) {
        gui.setValue(prog, i);   // 安全！会自动投递到主线程
        sleep(0.02);
        i = i + 1;
    }
});
worker.send("go");

gui.run();

Actor 线程中连续更新进度条，界面流畅无卡顿，完全线程安全。

8. 常见问题

Q: 中文显示为方框
A: 检查字体路径。确保 initGUIFont() 中加载了中文字体（如文泉驿、微软雅黑）。可在全局变量 g_font 初始化时指定系统存在的中文字体路径。

Q: 控件重叠
A: 如果使用布局，确保布局容器高度足够，且子控件的高度属性不为 0。标签高度至少 22，文本框/按钮必须传入高度参数。若手动布局，检查坐标是否冲突。

Q: 按钮回调中调用 return 会崩溃？
A: 不会。Mochi 的 ReturnException 已被 GUI 主循环安全捕获，可以正常使用 return 提前退出回调。

Q: 如何关闭窗口并继续执行脚本？
A: 调用 gui.quit() 会退出事件循环，gui.run() 返回后继续执行后续代码。
