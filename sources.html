🔍 Mochi 解释器源码解析

深入理解动态语言 Mochi 的 C++17 实现

📌 总体架构

Mochi 解释器遵循经典的 词法分析 → 语法分析 → 字节码执行 流水线，但由于动态语言特性，它采用 AST 树遍历解释 而非编译为字节码。整个系统由以下核心模块组成：

┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐ │ Lexer │──▶ │ Parser │──▶ │ AST │──▶ │Interpreter│ │ (词法) │ │ (语法) │ │ (抽象树) │ │ (解释器) │ └──────────┘ └──────────┘ └──────────┘ └──────────┘ │ ▼ ┌─────────────────┐ │ 运行时对象 │ │ (MochiClass, │ │ MochiInstance, │ │ MochiFunction, │ │ MochiArray, │ │ MochiDict...) │ └─────────────────┘

外加 Environment 管理作用域、NativeFunction 提供内置功能、Actor 并发模型、FFILibrary 外部函数接口，以及动态库加载器。

📝 词法分析器 (Lexer)

词法分析器将源代码字符串转换为 Token 序列。核心枚举 TokenType 包含所有关键字、运算符和字面量类型。关键方法 scanTokens() 逐字符扫描，识别数字、字符串、标识符和注释。

enum class TokenType {
    CLASS, FN, LOC, IF, ELSE, WHILE, FOR, RETURN,
    SUPER, THIS, TRUE, FALSE, NIL, AND, OR,
    IN, TRY, CATCH, THROW, BREAK, CONTINUE,
    IDENTIFIER, NUMBER, STRING,
    PLUS, MINUS, STAR, SLASH, MOD, BANG, EQUAL, ...
};

class Lexer {
    std::string source;
    size_t start, current;
    int line;
    static const std::map<std::string, TokenType> keywords;
    std::vector<Token> scanTokens();
};

在 scanToken 中，遇到 / 时判断是单行注释、多行注释还是除法；遇到 " 调用 stringLiteral() 处理转义字符；遇到数字则解析浮点数。关键字通过静态 map 识别，如 {"loc", TokenType::LOC}。

🏗️ 语法分析器 (Parser)

Parser 采用递归下降分析，从 declaration() 开始，根据当前 token 类型分发到 classDeclaration、function、varDeclaration 或 statement。表达式解析采用经典的运算符优先级爬升（Pratt parsing 变体），通过 expression → assignment → orExpr → andExpr → equality ... 逐级下降。

class Parser {
    std::vector<Token> tokens;
    size_t current;

    StmtPtr declaration();   // 分发到类、函数、变量、语句
    StmtPtr statement();     // 处理控制流、异常、表达式语句
    ExprPtr expression();    // 入口，调用 assignment
    ExprPtr assignment();    // 处理 = 赋值（包括索引赋值）
    ExprPtr orExpr();        // or 运算符
    ExprPtr andExpr();       // and 运算符
    ExprPtr equality();      // == !=
    ExprPtr comparison();    // > < >= <=
    ExprPtr addition();      // + -
    ExprPtr multiplication();// * / %
    ExprPtr unary();         // ! -
    ExprPtr call();          // 函数调用、点号访问、索引访问
    ExprPtr primary();       // 字面量、变量、this、super、fn 表达式、数组、字典
};

表达式 AST 节点 采用多态设计，每种表达式类型一个结构体，继承自 Expr。例如 Binary 包含左表达式、运算符 token、右表达式。类似地，Stmt 基类下有 IfStmt、WhileStmt、ForStmt、ForInStmt、ReturnStmt、ClassStmt 等。

特殊设计：Super 表达式在解析时固定父类方法，在解释时通过环境中的 super 变量找到父类。

🌳 AST 节点概览

类别节点说明
表达式Binary, Unary, Literal, Variable, Assign, Call, Get, Set, Super, This, FunctionExpr, ArrayLiteral, DictLiteral, IndexGet, IndexSet覆盖所有表达式
语句ExpressionStmt, PrintStmt, VarStmt, BlockStmt, IfStmt, WhileStmt, ForStmt, ForInStmt, ReturnStmt, ClassStmt, FunctionStmt, TryStmt, ThrowStmt, BreakStmt, ContinueStmt, EmptyStmt控制流与声明

所有节点通过 std::unique_ptr 管理，确保内存安全。

⚡ 解释器 (Interpreter)

Interpreter 类执行 AST 遍历。它维护一个 Environment 链（用于作用域）和全局环境 globals。核心方法 interpret() 遍历顶层语句列表，调用 execute() 执行每条语句。

class Interpreter : public std::enable_shared_from_this<Interpreter> {
    std::shared_ptr<Environment> globals;
    std::shared_ptr<Environment> environment;
    std::map<std::string, Value> modules;
    std::vector<void*> loadedHandles;

    void interpret(const std::vector<StmtPtr>& statements);
    void execute(const Stmt& stmt);
    Value evaluate(const Expr& expr);
    Value callFunction(std::shared_ptr<MochiFunction> func, const std::vector<Value>& args);
    std::shared_ptr<MochiFunction> bindFunction(...);
};

表达式求值 通过 evaluate() 的多态分发实现。例如 Binary 节点会根据运算符执行相应操作，并支持运算符重载（若左操作数是实例且有 __add__ 等方法）。Call 节点会提取 callee，若是类则创建实例并调用 init，若是函数则调用 callFunction。

return / break / continue 通过异常实现：抛出 ReturnException、BreakException、ContinueException，在对应循环或函数调用处捕获。

错误处理：throw 语句抛出 ThrowException，try/catch 捕获 ThrowException 和标准 C++ 异常，并将异常值绑定到 catch 变量。

🧩 运行时对象

所有值类型为 using Value = std::any，可以存储基础类型、shared_ptr 指向的各类对象。

struct MochiFunction {
    std::string name;
    std::vector<std::string> params;
    std::vector<StmtPtr> body;
    std::shared_ptr<Environment> closure;
};

struct MochiClass {
    std::string name;
    std::shared_ptr<MochiClass> superClass;
    std::map<std::string, std::shared_ptr<MochiFunction>> methods;
    std::shared_ptr<MochiFunction> findMethod(const std::string&);
};

struct MochiInstance {
    std::shared_ptr<MochiClass> klass;
    std::map<std::string, Value> fields;
    std::shared_ptr<Environment> globals; // 用于析构
};

struct MochiArray { std::vector<Value> elements; };
struct MochiDict { std::map<std::string, Value> entries; };

类继承通过 superClass 指针实现，方法查找沿原型链向上。实例字段存储在 fields 中，方法则从类中查询。为了支持跨线程自动析构，实例保存了全局环境引用，删除器通过临时解释器调用 destroy()。

🛠️ 内置函数与标准库

Interpreter::initBuiltins() 注册全局函数如 print、type、str、num、len、input、error、abs、min、max、random、sleep、clone、range 等。它们都是 NativeFunction，内部通过 lambda 捕获与 C++ 交互。

标准库模块 通过 registerBuiltinModules() 注册，每个模块返回一个 MochiDict。当执行 import("fs") 时，解释器优先查找内置模块表，找到则调用对应的工厂函数返回模块字典并缓存。类似地，math、json、random、string、array、dict 均以此方式实现。

builtinModules["fs"] = [this]() -> Value {
    auto mod = std::make_shared<MochiDict>();
    // read, write, append, exists, remove, listdir ...
    return mod;
};

GUI库基于SDL2编写，需要sdl2 2.32.8和sdl2_ttf 2.24.0其余sdl2源代码已涵盖在sources目录内

⚡ Actor 并发模型

每个 Actor 拥有独立的 MochiActor 对象和一条消息队列。创建时，spawn 函数构造 MochiActor，启动 std::thread 处理消息循环。消息可以是普通值或 AskContext（用于请求-响应）。reply 函数通过线程局部变量 currentAsk 将结果写回 Future。每个 Actor 线程拥有独立的 Interpreter 实例（共享全局环境），确保线程安全。

struct MochiActor {
    std::function<void(Value)> processMessage;
    std::queue<Value> mailbox;
    std::mutex mtx;
    std::condition_variable cv;
    std::thread worker;
    void send(Value msg);
    std::shared_ptr<MochiFuture> ask(Value payload);
    void close();
    void join();
};

MochiFuture 通过条件变量阻塞 get()，在 setValue 时通知。Actor 析构时自动 join 线程。

🔌 FFI 外部函数接口

FFILibrary 类维护两个映射表：nativeFuncs（C++ 函数注册）和 exportedFuncs（Mochi 导出函数）。通过 getFFI() 获取实例，调用 add 注册 C++ 函数。在 Mochi 中通过 ffi.call("name", ...) 调用，ffi.export("name", func) 导出。

class FFILibrary {
    std::map<std::string, std::function<Value(Interpreter*, const std::vector<Value>&)>> nativeFuncs;
    std::map<std::string, std::shared_ptr<MochiFunction>> exportedFuncs;
    void add(const std::string& name, auto func);
    Value call(const std::string& name, const std::vector<Value>& args);
    void exportFunc(const std::string& name, std::shared_ptr<MochiFunction>);
    Value callMochi(const std::string& name, const std::vector<Value>& args);
};

📦 动态库模块加载

import("name.mochimod") 通过 dlopen/LoadLibrary 加载动态库，并调用 mochi_module_init 获取模块字典。第三方开发者只需包含 mochi_api.h，实现该函数即可。解释器在 loadedHandles 中保存句柄，但不手动卸载，由操作系统在进程退出时回收。

// 第三方模块示例
extern "C" Value mochi_module_init(Interpreter* interp) {
    auto mod = makeDict();
    // 注册函数...
    return mod;
}

🌐 环境与作用域

Environment 类维护符号表 values 和指向外层环境的指针 enclosing。变量查找沿链递归。每个函数调用创建新环境，从而实现闭包。内置函数和全局变量存储在 globals 中，作为所有作用域的根。

📖 总结

Mochi 解释器展示了如何用C++17 代码构建一门功能丰富的动态语言。其设计强调可扩展性：内置模块、FFI、动态库加载使语言能力不受限。Actor 模型赋予原生的并发支持，而自动析构和深拷贝则提升了易用性。

这份文档仅概述核心思路，完整的实现细节请参考 Mochi 源码。希望它能帮助你快速掌握 Mochi 的内部世界，并为你的二次开发或第三方扩展提供指引。
